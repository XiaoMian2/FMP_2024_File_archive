4.List three advantages and three disadvantages of HTTP.
优点:
无状态的，每一个请求都是相互独立的，默认不需要保存上下文的信息。
    请求应答，有来有回。
    HTTP可以轻松实现集群化，扩展性能。
    缺点:
HTTP是不安全的，无法验证通信双方的身份，也不能判断报文是否被窜改。
    为了传输一个网页，一个HTTP需要创建多个连接。这会导致连接中的管理开销。
    即使 HTTP 接收到它需要的所有数据，客户端也不会采取措施关闭连接。因此，在此期间，服务器将不存在。
    

5.What authentication and web security did you use? Explain why you selected these methods.
用户可以通过输入他们的用户名和密码的方式实现，这允许系统确认他们的身份。
    网站认证过程的工作原理是将用户的凭证（用户名和密码）与文件中的凭证进行比较。
    这样可以确认用户的身份，保障用户的数据安全。
    用户在表单中输入信息时，需要确保用户没有非法或影响网站安全的输入。
    

6.Why was session management so important for the auto logout function? 
How is this different to a stateless web page?
session可以检查未操作的时间，长时间未操作就会自动登出，可以防止用户信息泄漏。
    无状态网页简单讲，就是不能识别用户，无法保存用户的各种信息。
    无状态页面上，每次都要在交互场景请求中带上上一次的请求信息，如账号、密码。
    

7.What HTTP protocol did you use for the server-side application.
Laragon中，打开浏览器访问默认站点http://localhost 就能够访问网页文件。
    提供构建现代Web应用程序所需的一切软件。
    GET和POST
GET和POST是HTTP请求服务器的两种基本方法.。
    GET对值的长度有限制，一般为255个字符。POST对值的长度没有限制，因为它们是通过HTTP的主体提交的。
    当点击网页上的链接或者通过在浏览器的地址栏输入网址来浏览网页的，使用的都是GET方式。
    

8.What data storage requirements did you need to consider?
采用了mysql数据库，它是关系型数据库，是开放源码，很多网站的开发都选择 MySQL 作为网站数据库。
    数据库中需要保存用户的用户名、密码、国家等数据。
    网站要保护数据安全，遵守数据保护法。
    对于重要数据避免明文传输和存储，可以进行加密。
